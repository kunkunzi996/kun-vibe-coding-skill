# 10 Codex Safe Construction：安全施工

用于所有会改代码、配置、文档、部署文件的任务。

## 开工前必须说明

- 当前目录。
- 当前分支。
- 本轮只做什么。
- 本轮不做什么。
- 预计修改哪些文件。

## 施工顺序

1. 先读项目结构和相关文件。
2. 再给最小施工计划。
3. 每次只做一个小任务。
4. 只修改直接相关文件。
5. 不乱格式化。
6. 不隐藏重构。
7. 不乱装依赖。
8. 不批量删除文件。
9. 不擅自改数据库结构。
10. 不擅自改部署方式。
11. 不擅自暴露公网。
12. 不擅自修改密钥、环境变量或凭证。

## 高风险动作必须暂停确认

- 安装新依赖。
- 改架构。
- 改数据库。
- 改 API 协议。
- 改目录结构。
- 替换核心依赖。
- 上传部署。
- 开公网隧道。
- 删除文件。
- 修改密钥或环境变量。

## 小步提交原则

一次只完成一个可验收的小目标。

不要把功能开发、UI 美化、重构、依赖升级、部署改动混在同一个任务里。
