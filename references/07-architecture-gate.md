# 07 Architecture Gate：架构门

架构门只在必要时使用。不要把每个小修改都变成架构讨论。

## 使用时机

开启架构门：

- 新项目
- 大功能
- 多页面 / 多模块功能
- 涉及数据库
- 涉及 API
- 涉及登录权限
- 涉及部署
- 涉及爬虫 / 定时任务 / 第三方服务
- 可能改变目录结构、数据模型、接口协议、核心依赖

跳过架构门：

- 小 UI 修改
- 文案修改
- 单文件小修复
- 明确的 bug 小修

- 模块 A：负责什么 / 不负责什么

数据从哪里来 → 在哪里处理 → 存在哪里 → 在哪里展示

## 核心数据模型
- 实体：字段、关系、说明

## API 或模块边界
- 前端如何调用后端
- 模块之间如何通信

## 关键规则

架构设计完成后，只拆出一个小 Task Spec 给 Codex 执行。

如果任务需要改变现有架构、目录结构、数据库、API、部署方式、核心依赖或模块边界，必须先暂停并让用户确认。
