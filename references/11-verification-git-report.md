# 11 Verification / Git / Report：验收、保存、汇报

## 真实验收

不要只根据代码阅读就说“完成”。能跑真实路径就必须跑。

### 前端 / UI

- 页面是否能打开
- 样式是否正常
- 按钮是否可点击
- 移动端是否有明显问题

### 数据功能

- 能否新增
- 能否读取
- 能否修改
- 能否删除
- 刷新后数据是否还在

### Bug 修复

- 原 bug 路径是否修好
- 附近正常流程是否仍然可用

### 部署

- 本地能否启动
- build 是否通过
- 环境变量是否完整
- 数据是否持久化
- 公网地址是否可访问

### 架构调整

- 旧入口是否还能用
- 路由是否正常
- API 是否兼容
- 导入路径是否正确
- 旧功能是否受影响

## 无法验收时必须说明

- 哪些验收没跑
- 为什么没跑
- 用户应该如何手动检查

## Git 保存规则

完成并验收后：

```bash
git status
git add .
git commit -m "简洁说明本次改动"
git push
```

如果没有远程仓库，只 commit，不假装 push。

如果当前目录不是 Git 仓库，直接说明。
