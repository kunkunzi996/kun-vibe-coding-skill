# 05 AI-SDD：AI 可执行项目规格书

AI-SDD 是本流程的主规格文档。

它不是传统长 PRD，也不是重型技术设计文档。它的目标是让 AI 知道：这个产品应该如何被拆成页面、模块、数据、接口、目录和验收标准。

## 使用时机

- Product Brief 和 MVP 已确定后。
- Pre-Coding Gate 确认需要执行规格时。
- Codex 正式开工前。
- 已有项目新增大功能前。
- 需要多人 / 多轮 AI 协作时。

## 变更规格原则

已有项目新增大功能时，不要重写完整 AI-SDD。只补充：

- 本次变更目标。
- 影响页面。
- 影响数据模型。
- 影响 API / 模块边界。
- 回归验收清单。
- 是否需要架构门。
- 是否需要 Computer-Use E2E。
