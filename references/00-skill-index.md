## 自然语言触发规则

### 用户说“新项目 / 想法 / 开工”

优先启用：

1. 想法压力测试
2. 开工规格
3. 产品简述
4. 执行规格
5. 必要时设计门 / 架构门

### 用户说“继续开发 / 接着做 / 加功能”

优先启用：

1. PROJECT_STATE
2. 单次任务
3. 测试门（按需）
4. 安全施工
5. 真实用户验收（按需）
6. 保存汇报

### 用户说“修 bug / 不对 / 报错”

优先启用：

1. 单次任务
2. 测试门
3. 安全施工
4. 真实用户验收（按需）
5. 保存汇报

### 用户说“上线 / 部署 / 公网 / 服务器 / Railway / Docker”

优先启用：

1. 单次任务
2. 架构门
3. 安全施工
4. 真实用户验收
5. 保存汇报

### 用户说“验收 / 跑通 / 像真实用户一样测”

优先启用：

1. 测试门
2. 真实用户验收
3. 保存汇报

## 跳过规则

- 已进入研发中期：跳过想法压力测试、开工规格、完整 Product Brief。
- 只是小 UI / 文案：跳过开工规格、执行规格、架构门。
- Bug 修复：跳过产品定位，先写复现路径。
- 只是保存：先检查验收和 Git 状态，不要重新规划。
- 用户明确说“不要重写 PRD / 不要重做规格”：必须遵守。
