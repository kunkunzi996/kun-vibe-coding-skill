# 07 Design Gate：设计基调门

用于开发前确定页面布局和视觉方向，避免 Codex 做出“能用但难看”的界面。

## 使用时机

- 新项目有前端页面。
- 仪表盘 / 工具页 / 管理后台 / 记录系统。
- 用户体验明显影响使用意愿。
- 用户明确要求 UI 或视觉风格。

## 不必使用

- 纯后端脚本。
- 命令行工具。
- 小 bug 修复。
- 简单文案修改。

## 反高保真沉迷规则

设计门不是让用户无限打磨高保真图。

如果出现以下情况：

- 核心功能尚未验证。
- MVP 尚未跑通。
- 致命卡点尚未解决。
- 用户持续要求优化视觉、配色、logo、动效、卡片样式。

必须提醒：

> 当前阶段不要继续扩大设计细节。先回到 MVP，验证核心功能能不能跑通。视觉细节放到 v0.2 / v0.3。

## 检查问题

- 产品气质是什么？工具型、仪表盘型、内容型、记录型？
- 主色调和整体风格是什么？
- 每个页面第一眼应该看到什么？
- 哪些信息需要突出？
- 哪些信息需要弱化？
- 是否需要移动端适配？
- 是否需要空状态、加载状态、错误状态？
