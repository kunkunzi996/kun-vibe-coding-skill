# 02 Spec Start：开工规格（qiaomu-ai-prd inspired）

这是规格生成模块。

它吸收 qiaomu-ai-prd 的方法：把一句话产品想法整理成开发者和 AI 都能执行的规格草案，而不是生成传统冗长 PRD。

## 使用时机

- 用户提出一个新项目想法。
- 用户说“帮我开工”。
- 用户还没有 Product Brief / AI-SDD。
- 用户需要把模糊想法交给 Codex 前先整理。
- 已有项目新增大功能，但需要先生成变更规格。

研发中期不要重复使用，除非用户明确要求重新整理规格。

## 核心原则

1. 主动做产品判断，不要把所有模糊点都推回给用户。
2. 输出要能给实现 Agent 直接读懂。
3. 区分“硬约束 / 推荐默认 / 发挥空间”。
4. 页面要尽量用 ASCII 图表达真实布局，而不是空泛描述。
5. 数据模型要有字段说明和版本字段。
6. 性能指标要数字化，不能只写“快、流畅、稳定”。
7. 必须有验收剧本和开发者交接说明。
8. 必须诚实写出已知未知项。

## 默认策略

如果用户没有明确说明，优先采用：

- 单用户优先。
- Web App 优先，除非用户明确要小程序、桌面或移动端。
- 本地存储 / 轻量数据库优先，除非需要多设备同步、分享、权限、支付。
- 先做核心闭环，不做用户系统、复杂权限、订阅、复杂通知。
- 先做可用界面，不做过度动效和复杂设计系统。

## 注意

本模块生成的是“规格草案”，不是最终施工命令。

生成后通常要继续进入：

- `03-pre-coding-gate.md`，准备施工前检查
- Product Brief 定稿
- AI-SDD 定稿
- Architecture Gate，如涉及架构
- Task Spec，进入具体施工
